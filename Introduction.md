# Introduction to all topics

## HLD->

### Horizontal Scaling
Adding more machines/instances to handle increased load by distributing traffic across multiple servers.

### Vertical Scaling
Increasing the power (CPU, RAM) of a single server to handle more load.

### Distributed Systems (Partitions)
Splitting data or workload across multiple machines (partitions) to improve performance and reliability.

### Microservices Architecture
Breaking an application into small, independent services that communicate with each other.

### Load Balancer
A system that distributes incoming requests across multiple servers to ensure no single server is overloaded.

### Backup Servers
Standby servers that take over when the main server fails to ensure system availability.

### Pre-processing using CRON Jobs
Running scheduled background tasks (like cleanup or data processing) at fixed time intervals.

### Decoupling
Designing components to work independently so changes in one part don’t affect others much.

### Logging and Metrics Calculation
Collecting system data for monitoring and analysis:
1. Analytics – Understanding user behavior and trends  
2. Auditing – Tracking actions for security/compliance  
3. Reporting – Generating summaries and insights  
4. ML – Using data to train models and make predictions  

### Scalability and Extensibility
Scalability is handling growth in users/data, while extensibility is easily adding new features.
